System Design Document

Daily / Unlimited

How players discover, choose, and move between play types across every game mode, every surface, and every screen.
April 2026 · Wordle Global
01

The Problem

Daily and unlimited are tangled into the mode list

Today the sidebar lists 10 game modes as flat items. Five of those are multi-board variants (Dordle, Quordle, Octordle, Sedecordle, Duotrigordle) that differ only in board count. "Daily Puzzle" and "Unlimited" are treated as separate modes, but they're actually play types of the same Classic game.

As we add daily variants to multi-board modes and unlimited to Semantic, the sidebar would double in length. Worse: the conceptual model is wrong. Dordle Daily and Dordle Unlimited aren't two different games — they're the same game with different word selection. The navigation should reflect that.

Core insight: the game mode matrix has two independent axes — board type (Classic, Dordle, Quordle, Semantic, Speed) and play type (Daily, Unlimited). The current sidebar conflates them into a flat list, making both axes harder to navigate.

What needs to work: a player who finishes daily Quordle should discover unlimited Quordle exists. A player browsing modes shouldn't see 14 items. The archive should show daily history for every mode. Streaks should be per-mode-per-play-type. And all of this should feel like 6 lines of navigation, not 14.

03

Design Principles

Rules that resolve ambiguity

1. Play type is chosen before the game, not during.
No in-game toggle. No config bar. The subtitle says which type you're in. That's it.

2. Daily is the default. Unlimited is the upsell.
Clicking any mode card or sidebar item goes to daily (when available). Unlimited is discovered through post-game CTAs and sidebar expansion.

3. Post-game is the highest-conversion entry point.
After finishing daily Quordle, the stats modal says "Play Unlimited Quordle." After unlimited, it nudges "Try today's daily." This cross-pollination does more work than any sidebar redesign.

4. Streaks are daily-only. Unlimited has no streak.
Streaks represent consecutive calendar days. Unlimited games happen on demand — they have play count, win rate, but no streak.

5. The archive is daily-only.
The word archive shows past daily words for each mode. Unlimited games don't have "past words" — they're random.

04 / Surface

In-Game

A label, not a control

The header subtitle tells the player which play type they're in. "Dordle #142" = daily (the number implies it). "Dordle · Unlimited" = unlimited (accent color, no number). Nothing interactive. No toggle. The player chose before entering.

05 / Surface

Sidebar

5 items, not 10. Multi-board is bundled.

Multi-board modes (Dordle through Duotrigordle) collapse into one "Multi-Board" row. Tapping it expands to show board counts. Modes with both play types (Multi-Board, Semantic) show daily/unlimited sub-items on expand. Single-type modes (Daily Puzzle, Unlimited, Speed) navigate directly.

Compare to the original Screen 02 sidebar which had 10 flat items and no daily/unlimited distinction. Also compare to Option C (Mode-First Sidebar) which explored a similar hierarchy but without bundling multi-board.

06 / Surface

Post-Game

The highest-conversion moment

The stats modal after game-over is where cross-pollination happens. After daily: "Play Unlimited" in accent. After unlimited: "Try Today's Daily" in green. This does more discovery work than any sidebar or homepage redesign.

Compare to Screen 03 (End of Game) which had a basic "Play Unlimited" secondary button. The new design is contextual — different CTAs depending on what just ended.

08 / System

Streaks

Daily-only. Per-mode. Visible in sidebar and post-game.

Streaks count consecutive calendar days of playing the daily variant. Each mode has its own streak: your Classic daily streak is independent of your Dordle daily streak. Unlimited has no streak — it has play count and win rate, but no "consecutive days" concept.

The streak badge in the header shows the highest active streak across all modes. The sidebar shows per-mode streaks next to each daily item. The streak dropdown (see Streak exploration) shows the full breakdown.

Play TypeStreakWin %Games PlayedBest
Daily🔥 Yes — consecutive daysYesYesYes (max streak)
Unlimited— Not trackedYesYes (total rounds)—
Edge case: if a player completes daily Dordle AND plays 3 unlimited Dordle rounds in the same day, the stats page shows: Daily — 7-day streak, 42 played, 88% win. Unlimited — 18 played, 72% win. Separate sections, separate stats.
09 / System

Archive

Daily-only. Per-mode. Browsable history.

The archive page (/en/words) currently shows past Classic daily words. With multi-mode dailies, the archive gains a mode filter at the top: Classic, Dordle, Quordle, etc. Each mode's archive shows its daily history. Unlimited games don't appear in the archive — they're ephemeral.

10 / System

Routing & URLs

One URL per mode. Play type is a query param.

Daily and unlimited share 100% of the game UI, logic, and keyboard. They differ only in word selection. Separate routes would duplicate SEO and split analytics. Instead:

URLResult
/en/dordleDaily Dordle (default)
/en/dordle?play=unlimitedUnlimited Dordle
/en/semanticDaily Semantic (default)
/en/semantic?play=unlimitedUnlimited Semantic
/en/speedDaily Speed (default — same word sequence for all)
/en/speed?play=unlimitedUnlimited Speed (random sequence)
/enDaily Classic (no change)
/en/unlimitedUnlimited Classic (no change)

The ?play=unlimited param triggers a soft reset — no page navigation, just different word selection. Last choice is saved to localStorage per mode so returning players land on their preferred type. Classic keeps its separate /unlimited route for backward compatibility and SEO (it's already indexed).

What We're NOT Doing

No config bar. F3 explored a persistent config bar with Daily/Unlimited/Custom pills. Rejected — too much UI for a binary choice.

No in-game toggle. The Option B approach of toggling play type mid-game was confusing. You pick before you play.

No mode picker modal. Screen 09 was a full-screen mode picker. Unnecessary when the sidebar already handles mode selection.

No separate sidebar items for daily/unlimited. That was the Card Split approach — rejected because it doubles the sidebar.
